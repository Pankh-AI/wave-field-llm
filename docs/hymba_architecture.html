Wave-Hymba: Parallel Head Architecture

6 wave heads (O(n log n)) + 2 attention heads (O(n²)) running in parallel within every layer

The Key Idea

Instead of alternating wave and attention LAYERS, run both mechanisms as parallel HEADS within every layer. Every layer gets content-dependent routing.

V4.6.0: Layer-level hybrid

L0: all 8 heads wave
L1: all 8 heads wave
L2: all 8 heads wave
L3: all 8 heads attention ← only content routing
L4: all 8 heads wave
L5: all 8 heads wave
L6: all 8 heads wave
L7: all 8 heads wave
Content routing at 1/8 layers only

V4.8: Hymba-style parallel heads NEW

L0: 6 wave + 2 attn in parallel
L1: 6 wave + 2 attn in parallel
L2: 6 wave + 2 attn in parallel
L3: 6 wave + 2 attn in parallel
L4: 6 wave + 2 attn in parallel
L5: 6 wave + 2 attn in parallel
L6: 6 wave + 2 attn in parallel
L7: 6 wave + 2 attn in parallel
Content routing at EVERY layer
⚡
Why parallel > alternating: NVIDIA's Hymba ablation showed parallel heads beat sequential by +4.74% recall accuracy with 13% better throughput. Each layer sees both distance-based patterns (wave) AND content-based routing (attention) simultaneously.

Shared QKVG Projection

One matmul produces Q, K, V, Gate for ALL 8 heads [Q, K, V, Gate] = LayerNorm(x) @ Wqkvg + b

Wqkvg: (384, 1536)   b: (1536,)

// 384 × 4 = 1536 (Q:384 + K:384 + V:384 + Gate:384)
// Each 384 = 8 heads × 48 dims/head

FLOPs: 2 × 512 × 384 × 1536 = 604M
Params: 384 × 1536 + 1536 = 591,360

// After projection, split by head index:
Q_wave = Q[:, :6, :, :] // (B, 6, 512, 48)
Q_attn = Q[:, 6:, :, :] // (B, 2, 512, 48)
// Same split for K, V

Wave Field Path (6 heads)

Step 1: Learned Feature Maps (Hedgehog) Q' = ReLU(W2 · GELU(W1 · Q_wave)) + ε
K' = ReLU(W2 · GELU(W1 · K_wave)) + ε

// W1, W2: (48, 48) identity-init, per head
// 2 maps × 2 layers × (48×48 + 48) = 9,408 params
FLOPs: 2 maps × 2 layers × 512 × 6 × 48² = 56M
Step 2: Deposit + Scatter depositj[d] = K'j[d] × Vj[d] // element-wise
field[p(j)] += depositj // scatter_add, stride=1

field shape: (B, 6, 512, 48)
FLOPs: ~0.3M (scatter is index ops, not matmul)
Step 3: FFT Convolution with Wave Kernel // Kernel per head: k(t) = e-αt cos(ωt + φ) for t≥0
// Z-transform: closed-form FFT, no materialization
// + SpectralGate: modulates kernel based on token 0's Q

field_fft = rfft(field, n=2048) // 4x pad for SpectralGate
kernel_fft = Z_transform(α, ω, φ) × (1 + gate(Q0))
convolved = irfft(field_fft × kernel_fft)[:512]

// 3 FFTs × 6 heads × 48 dims × 2048 × log(2048)
FLOPs: ~22M (with 4x padding for SpectralGate)
Kernel params: 3 × 6 = 18 (α, ω, φ per head)
SpectralGate params: ~98K (MLP: 6×48 → 288 → 6×32)
Step 4: Coupling + Gather + Cross-dim coupledh = ∑h' softmax(C)h,h' · fieldh'
gatheredi = coupled[p(i)] // gather at token positions
output = Wcross(Q' ⊙ gathered) // (48,48) linear, identity-init

C: (6, 6) = 36 params
Wcross: (48, 48) = 2,304 params
FLOPs: ~18M

wave_output: (B, 6, 512, 48)

Standard Attention Path (2 heads)

Scaled Dot-Product Attention (PyTorch SDPA) scores = Q_attn @ K_attnT / √48

// scores shape: (B, 2, 512, 512)
// This is the O(n²) part: 512 × 512 = 262K weights per head

attn = softmax(scores + causal_mask)
attn_output = attn @ V_attn

// attn_output: (B, 2, 512, 48)

FLOPs:
  Q@KT: 2 × 512 × 512 × 48 × 2 = 50M
  attn@V: 2 × 512 × 512 × 48 × 2 = 50M
  Total: 100M

Extra params: 0 (Q, K, V come from shared QKVG projection)
≡
2 attention heads use F.scaled_dot_product_attention (PyTorch 2.0+). This auto-selects FlashAttention-2 on GPU. Much faster than manual Q@K softmax@V.

Merge + Gate + FFN

Per-branch RMSNorm then combine (Hymba-style) // Wave and attn outputs have different magnitude distributions
// RMSNorm each independently before merging

wave_normed = RMSNormwave(wave_output) // (B, 6, 512, 48)
attn_normed = RMSNormattn(attn_output) // (B, 2, 512, 48)
combined = concat(wave_normed, attn_normed) // (B, 8, 512, 48)

// Content-dependent gating (from QKVG projection)
output = σ(Gate) ⊙ combined // Gate bias=2.0, starts open
output = reshape(output) // (B, 512, 384)

RMSNorm params: 2 × 48 = 96
FLOPs: ~0.2M
Output Projection + Residual x = x + Dropout(Wout @ output + b)
Wout: (384, 384) = 147,840 params
FLOPs: 2 × 512 × 384² = 151M
Feed-Forward Network x = x + Wdown · GELU(Wup · LayerNorm(x) + bup) + bdown
Wup: (384, 1536)   Wdown: (1536, 384)
Params: 1,181,568
FLOPs: 2 × 512 × 384 × 1536 × 2 = 1,208M

Attention FLOPs: Hymba vs V4.6.0

V4.6.0: 1 full attn layer at L3

7 wave layers: 0 attn FLOPs
1 attn layer: 402M FLOPs (8 heads)
Total attn: 402M

Hymba: 2 attn heads × 8 layers

8 layers × 100M = 800M FLOPs
Total attn: 800M (2× more)
But: spread across ALL layers
Hymba uses 2× more attention FLOPs but distributes them across every layer. The question is whether "a little routing everywhere" beats "a lot of routing at one point." NVIDIA's data says yes: parallel > concentrated.
